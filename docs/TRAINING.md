## 1. Preparation

### Download Our Processed MERLIN Training Data (masks)

- **Quick start:** This repository already provides several demo cases in `data/merlin_data_train_demo/`. These demo cases are intended only for verifying that the training pipeline can run successfully, so downloading the full dataset is **not required** at this stage.
- **Or train the model on the full dataset:** Please download the MERLIN dataset from the [Stanford AIMI Shared Datasets](https://stanfordaimi.azurewebsites.net/datasets/60b9c7ff-877b-48ce-96c3-0194c8205c40), and our processed MERLIN training masks from Hugging Face.
  - Due to upload size limitations, the full training masks are provided in three parts under `merlin_data_train_full/resized_masks`: `part00`, `part01`, and `part02`. Before training, merge the contents of these three parts into a single folder.
  - Using `RADAR_train/preprocess_code/process_img.py` to resize original merlin image to `[1, 1, 5]`.
  - After preparing the full dataset, update the `vis_root` path at line 79 in `RADAR_train/lavis/datasets/datasets/caption_datasets.py` so that it points to the merged training data folder.

## 2. Training

For quick start:

```bash
cd RADAR_train
python train.py
```

For multi-GPU training:

```bash
cd RADAR_train
torchrun --nproc_per_node=8 train.py
```

> **Note:** We trained the model using 24 GPUs (A100 or H20) with a total batch size of 48.

### Configure RADAR+ Global Alignment

Set `radar_plus` to `False` in `radar_config.yaml` to disable RADAR+'s global alignment module. The default value is `True`.

### Train from Scratch vs. Fine-tuning from RADAR Checkpoint

Set `radar_ft` to `True` in `radar_config.yaml` to enable fine-tuning from the pretrained RADAR checkpoint. The default value is `False` (training from scratch).

- We provided a RADAR+'s checkpoint (`checkpoint_radar_plus.pth`), which is one of our reproduced experiments trained from scratch.
- We also provided a fine-tuned version (`checkpoint_radar_plus_finetuned_on_merlin.pth`).

## 3. Inference

> **Note:** Inference can be run on a single GPU or multiple GPUs (A100 or H20). Multi-GPU is recommended to speed up large-scale inference.

- We use the official prompts released by MERLIN during inference.
- Before running inference, ensure that the [original MERLIN data](https://stanfordaimi.azurewebsites.net/datasets/60b9c7ff-877b-48ce-96c3-0194c8205c40) has been downloaded, and update the data path:

```python
Modify the img_dir in infer_merlin_anatomy.py and infer_merlin_whole.py
img_dir = "/your_path/merlin_data"
```

### Anatomy-Level Findings

Single-GPU:

```bash
cd RADAR_train
python infer_merlin_anatomy.py
```

Multi-GPU (recommended):

```bash
cd RADAR_train
torchrun --nproc_per_node=8 infer_merlin_anatomy.py
```

### Whole-Image Findings

Single-GPU:

```bash
cd RADAR_train
python infer_merlin_whole.py
```

Multi-GPU (recommended):

```bash
cd RADAR_train
torchrun --nproc_per_node=8 infer_merlin_whole.py
```

The inference results will be saved as CSV files.

## 4. Evaluation

Compute performance metrics using the inference CSV and the label file:

```bash
cd RADAR_train
python calc_metrics.py
```

The table below summarizes the evaluation performance of RADAR+ under different training settings.

| RADAR+ | Trained from scratch on Merlin-CT-Train | 0.888         | /           | `ckpt/checkpoint_radar_plus.pth`                     |
